# users-app
App to authenticate users and calculate a value of a sum adding a fee based on one external service

## Usage
For using this app you should follow the next steps:

1. Download the [collection](https://www.getpostman.com/collections/94919faf38f8a10d9a7f).
2. Deploy the container on your docker environment
3. Run the application

## Security
The application provides an signUp and Login that response an Bearer Token with 1 day of expiration. This token must 
be used on each API request.

## Restrictions
The next 2 endpoints:
```
http://localhost:8080/fee/1/2?mock_error=false
```
Only will be able to 50 requests per day due to the mock external dependency restriction.

## Installation
Docker:
1. Pull the docker image
```bash
docker pull lazun/users:latest
```

2. Up the docker compose

```bash
docker-compose up
```

## Use cases

1. Register User: You can use the next endpoint:
    ```
    curl --location --request POST 'http://localhost:8080/user/signup' \
    --header 'Content-Type: application/json' \
    --data-raw '{
        "name": "testName",
        "email": "testName@gmail.com",
        "password": "testpsw"
    }'
    ```

2. Log in user: You can use the next endpoint: 
    ```
    curl --location --request POST 'http://localhost:8080/auth/login' \
    --header 'Content-Type: application/json' \
    --data-raw '{
        "name": "some5",
        "password": "somepsw"
    }'
    ```
    This endpoint give you a Bearer Token to make the next requests.
    
    Response example:

   ```json lines
    {
        "accessToken": "TOKEN.asdasd.ee-a-s-ddd",
        "tokenType": "Bearer"
    }
    ```

3. Add numbers with tax: This endpoint sum up two numbers adding a tax provided for an external service.
   Receive a parameter (mock_error). When this parameter is true, the endpoint consumes a mocked API that always returns an INTERNAL SERVER ERROR.
   When you just boot up the App and send mock_error=true the API will return an error, due to not having a value to 
   return. The API will retry 3 times (you can look it up in the logs)
   If you send the parameter mock_error=false the API will consume a mocked API that returns a value that will be cached by 30 minutes.
   
   Examples:
   Request:
    ```
   curl --location --request GET 'http://localhost:8080/fee/1/2?mock_error=true' \
   --header 'Authorization: Bearer YOUR_TOKEN'
    ```
   Response:
   ```json lines
   {
    "error": "INTERNAL_SERVER_ERROR",
    "message": "An error occurred",
    "status": 500
   }
   ```
   
   Request with mock_error=false:
   ```
   curl --location --request GET 'http://localhost:8080/fee/1/2?mock_error=false' \
   --header 'Authorization: Bearer YOUR_TOKEN'
    ```
   Response:
   ```json lines
   {
    "amount": 3.3000002
   }
   ```
